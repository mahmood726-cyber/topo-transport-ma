Topological Transportability for Meta-Analysis

When you transport a pooled effect to a new target population, positivity (overlap of effect-modifiers) is usually assumed. This capsule computes the persistent homology of the effect-modifier space and uses it as a geometry-aware positivity certificate — catching targets that fall in a hole of the evidence base, which a convex-hull check passes as “supported.” The transport estimate (effect-modifier meta-regression) and the topological support grade are independent layers.

3 · Method, in one paragraph

Represent the studies as a point cloud in effect-modifier space. The transported effect is the generalizability estimand  τ_target = β₀ + β′·E_target[x]  from a random-effects meta-regression (REML τ² via Fisher scoring). Independently, the Vietoris–Rips persistent homology of the cloud gives H0 (connected components → latent subpopulations the scalar I² cannot see) and H1 (loops → holes), with holes confirmed against sampling noise by a subsampling bottleneck band (Fasy et al. 2014). A target is graded GAP when the convex hull accepts it but it lies in a confirmed hole; GOLD/SILVER/BRONZE/NONE grade decreasing geometric support. Odds ratios are refused (non-collapsible: a transported OR has no population causal meaning); ratio-scale transports with baseline-risk SD > 0.1 trigger a risk-difference sensitivity flag.

Engines (persistent homology, REML meta-regression, support certificate) are reimplemented here in JavaScript and re-validated live in the Self-audit panel above; the Python package validates the same engines against GUDHI and metafor/statsmodels.
